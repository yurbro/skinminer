## Implementation notes

- Table prompt and structured LLM schema now emit `endpoints[]` with `kind`, `mean`, `sd`, `unit`, and `n_replicates`.
- Endpoint priority is deterministic: flux first, then cumulative amount, then permeability/partition/diffusion/lag/fraction/unknown.
- v1/v2/v3 policy checks select a cumulative amount endpoint when present, so paired flux+cumulative records remain compatible with strict amount policies.
- v4 still preserves PR-A effective-kind metadata and accepts unit-implied flux.
- Table LLM input/output cache is written to `outputs/v1.5_pr_c_llm_cache`.
- A deprecated Python-only `EndpointSpec` construction shim remains so out-of-scope text/figure modules can import and instantiate their old helper during this PR. It is not a `Record` field, and old JSON with top-level `endpoint` still fails validation.
